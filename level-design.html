Silent Passage Demo

Level Design Handoff

레벨디자이너는 에디터에서 레벨을 만들고, JSON으로 export한 뒤 Git 브랜치/PR로 공유합니다. 브라우저 localStorage 저장은 개인 임시 저장소입니다.

Open Level Editor Open Game Workflow README

1. Branch

레벨 작업은 기능 개발과 분리된 브랜치에서 시작합니다.

git checkout -b level/faceoff-checkpoint-pass-01

2. Build

에디터에서 레벨을 만들고, 중간 저장은 로컬 저장을 사용합니다.

http://localhost:4173/editor.html

3. Export

팀 공유용 결과물은 반드시 JSON으로 export해서 `levels/drafts/`에 둡니다.

levels/drafts/faceoff-checkpoint-01.v001.json

4. Validate

PR 전에 route/entrance 연결 오류를 검사합니다.

node scripts/validate-levels.mjs

Rules

  • `route exit`은 레벨 간 이동에 사용합니다.
  • `extraction exit`은 런 종료 지점에만 사용합니다.
  • `routeExit.toLevelId`는 실제 존재하는 레벨이어야 합니다.
  • `routeExit.toEntranceId`는 대상 레벨의 실제 entrance ID여야 합니다.
  • 기본 내장 레벨은 직접 삭제하지 않습니다.

Commit

git add levels/drafts/faceoff-checkpoint-01.v001.json
git commit -m "Add draft faceoff checkpoint level"
git push origin level/faceoff-checkpoint-pass-01
